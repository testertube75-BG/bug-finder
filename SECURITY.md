# Security Policy

## Authorized Use Only

BG Bug Scout is intended for systems you own or have explicit written permission to test. Do not use this project to attack, exploit, overload, bypass, or access systems outside your approved scope.

## Reporting Vulnerabilities

If you find a vulnerability in BG Bug Scout itself, please report it privately to the repository owner. Include:

- A short summary.
- Affected version or commit.
- Steps to reproduce.
- Expected and actual behavior.
- Security impact.

Do not include secrets, tokens, personal data, or third-party private information in public issues.

## Safe Testing Principles

- Stay inside program scope.
- Prefer low-impact validation.
- Stop if a test exposes private data.
- Follow the target program's disclosure rules.
- Keep reports private until the owner gives permission to publish.
